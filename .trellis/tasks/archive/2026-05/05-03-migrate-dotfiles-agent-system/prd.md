# PRD: Migrate Dotfiles Agent System To Trellis

## Goal

Make Trellis the source of truth for the dotfiles agent workflow and remove the
old duplicated Claude/Codex rule-file installation model.

## Requirements

- Track Trellis project state in `.trellis/` with Claude and Codex platform
  adapters generated by Trellis.
- Move durable agent rules from `claude/rules/*.md` and `codex/rules/*.md` into
  `.trellis/spec/agent/`.
- Keep global Claude and Codex entry files short. They should point agents to the
  Trellis workflow/spec locations instead of owning detailed rules directly.
- Update `scripts/restore.sh` so a restore installs Trellis global entry points,
  Trellis skills, Trellis agents, and removes obsolete rule symlinks.
- Enable Codex hooks in the rendered user-level Codex config so project-local
  Trellis hooks can run where installed.
- Update repository documentation and symlink map to describe the Trellis-based
  layout.

## Acceptance Criteria

- `git status --short --untracked-files=all` shows Trellis files, updated docs,
  and removal of old rule/agent sources, with no Python cache files.
- `python3 ./.trellis/scripts/task.py validate 05-03-migrate-dotfiles-agent-system`
  passes.
- `python3 ./.trellis/scripts/get_context.py --mode packages` lists the new
  dotfiles/agent specs.
- `bash -n scripts/restore.sh` passes.
- The rendered Codex config contains `[features] codex_hooks = true`.
- `restore.sh` no longer links `~/.claude/rules` or `~/.codex/rules`.

## Out Of Scope

- Running `scripts/restore.sh` against the live `$HOME`.
- Committing, pushing, or opening a pull request.
- Migrating unrelated editor, terminal, or Homebrew configuration.
